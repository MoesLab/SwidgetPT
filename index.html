Swidget-Control4 WiFi Driver Development Stages

Stage:{ A } 70% off all work
Starts by receiving hardware from Swidget, The base driver model will be initiated which will evolve to 3 major Host-bases used by 8 Inserts. In this stage major work will include Device Identification, Registration, and Configuration to get BASIC or SECURE the connection between C4 Controller and Swidget devices. By end of this stage, there will be 3 drivers that can accept Control Inserts.

ETD around Aug. 2021
 Stage:{ B }
Will continue Stage A, and the other 7 Inserts will be added to the parent drievrs. there will be test, and documentation process right after, to get the 3 up-scaled drivers ready for Certification in Stage C

ETD around Oct 2021

 Stage:{ C }
In this stage, All 3 drivers will be sent for Certification by C4.

ETD around Nov. 2021

Stage: { D } Will include advanced functionalities such as OTA firmware update from C4 controller, and Master-Manager Driver
